Overview
The Centinel Thin Client samples outline a template for processing transactions with the Centinel system. The samples provide code examples of how a merchant can integrate the different Centinel API messages into their ecommerce platform.
 
Centinel Configuration
Before using the Centinel Thin Client samples, there are a few configurations that must be set up by logging into the Centinel Merchant account that you will be using to run the samples:

  • A transaction password must be set on the Centinel Merchant account.
  • A payment initiative must be set up on the Centinel Merchant account for each payment type that you will be integrating.
    •

    NOTE: For additional details on setting up a transaction password or a specific payment initiative configuration, please reference the specific Centinel Integration Guide for that payment initiative.

     
    Samples Configuration
    Copy the Centinel Thin Client samples to a location on the web server. Before the samples can be used a few minor configuration values must be updated in the CentinelConfig file for each Payment Intiative that you will be using:

  • TransactionUrl - update this value to point to the appropriate Centinel system (this value will be https://centineltest.cardinalcommerce.com/maps/txns.asp for testing purposes unless otherwise specified).
  • ProcessorId - update this value with the processor id value assigned during the merchant registration process.
  • MerchantId - update this value with the merchant id value assigned during the merchant registration process.
  • TransactionPwd - update this value with the transaction password that you have set up on your Centinel merchant account.
  • MessageVersion - update this value with the message version for the Centinel API you are integrating (current version is 1.7).
  • Timeout - update this value with the communication timeout threshold with the Centinel server (Contact Cardinal Commerce for recommended values, if needed).
  • TermURL - update this value (when applicable) to reference the exact path to the ccAuthenticate page for the specific payment initiative code examples that you are using.
